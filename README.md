# Semantic convergence of DPs as quality predictor for stance

A study investigating whether the **semantic convergence of Differentiating Properties (DPs)**
generated by large language models can serve as a predictor of **stance** (polarization)
classification quality.

Working hypothesis: sets of DPs that are semantically **convergent** (SIM) should yield
different class-separation quality than **divergent** (DIF) sets. Convergence is measured via
**DVA** analysis (geometric analysis of the embedding space), while quality is measured by the
performance of an SVM classifier trained on the embeddings.

## Experimental domains

The experiment is run independently for two topical domains:

| Domain | Results prefix | Notebook | Input data |
|--------|----------------|----------|------------|
| Politics (US, Dem vs Rep) | `POLITYCZNE` | [extract_graph_politics_one.ipynb](extract_graph_politics_one.ipynb) | `documents/dem_platform_2063.json`, `documents/demo_repub_dataset*.json` |
| Health / vaccines | `ZDROWOTNE` | [extract_graph_vacciness_one.ipynb](extract_graph_vacciness_one.ipynb) | `documents/vacciness_dataset.json` |

## Pipeline

Each notebook implements the same flow (sections 0–6 in the notebook):

1. **Load documents** – source texts from `documents/*.json` loaded as
   `langchain.Document` objects.
2. **Generate DPs (Differentiating Properties)** – multi-agent extraction of semantic features
   using LLMs (Claude / Gemini / OpenAI) – module
   [`helpers/semantic_multi_agent_definitions.py`](helpers/semantic_multi_agent_definitions.py).
3. **Select DPs before simulation** – choose the **SIM** (convergent) and **DIF** (divergent)
   variants and the set size **N = 3 / 5 / 10** properties; alongside a **BASELINE** variant.
4. **Embeddings** – convert texts to vectors (OpenAI / Gemini / Voyage / local Ollama) –
   [`helpers/polarization_embedings.py`](helpers/polarization_embedings.py).
5. **DVA analysis** – geometric assessment of semantic convergence (cosine, projection diff,
   Spearman and Pearson correlations) – [`helpers/dva_group.py`](helpers/dva_group.py).
6. **Classification (SVM)** – semantic polarization / stance classification on the embeddings –
   [`helpers/svm_polarization.py`](helpers/svm_polarization.py).
7. **Reports** – confusion matrices and classification & DVA metrics written to the
   `results_*` directories.

## Result file naming convention

Files in `results_*` encode the experiment configuration, e.g.
`results_POLITYCZNE_SIM_5_ONE_openai_classification_metrics_results.txt`:

- **`POLITYCZNE` / `ZDROWOTNE`** – domain,
- **`BASELINE` / `SIM` / `DIF`** – DP set variant (baseline / convergent / divergent),
- **`3` / `5` / `10`** – DP set size,
- **`openai` / `gemini`** – model used for embeddings,
- **`_565` / `zewnetrzne_1000`** – larger / external evaluation set,
- **`_classification_metrics_results.txt`** – accuracy, precision, recall, F1,
- **`_dva_metrics.txt`** – DVA metrics (cosine, projection diff, Spearman, Pearson).

## Getting started

1. Fill in the API keys in [`helpers/all_keys.py`](helpers/all_keys.py)
   (`API_KEY_OPENAI`, `API_KEY_ANTROPIC`, `API_KEY_GEMINI`; optionally Voyage):

   ```python
   API_KEY_OPENAI   = "..."
   API_KEY_ANTROPIC = "..."
   API_KEY_GEMINI   = "..."
   ```

2. Install the dependencies:

   ```bash
   pip install -r requirements.txt
   ```

3. Open one of the notebooks (`extract_graph_politics_one.ipynb` or
   `extract_graph_vacciness_one.ipynb`) and run the cells in order. The notebooks import code
   from the `helpers/` package, so run them from the repository root.

> **Note:** `helpers/all_keys.py` ships with empty API keys — do not commit your real keys to
> the repository.

## Requirements

- Python 3.10+
- API access: OpenAI, Anthropic (Claude), Google Gemini (optionally Voyage AI or a local
  Ollama instance for embeddings).
